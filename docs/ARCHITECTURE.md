# Konfa

## Цель проекта

Konfa — приватный мессенджер для небольшой группы друзей.

## Основные требования

- React + Vite
- Feature-Sliced Design (FSD)
- Supabase
- Cloudflare Pages
- Без собственного backend
- Только текстовые сообщения
- End-to-end encryption (E2EE)
- Realtime через Supabase
- Хранение истории сообщений 180 дней
- Автоматическая очистка истории

## Архитектура

```text
React
  │
Cloudflare Pages
  │
Supabase
├── Auth
├── Postgres
├── Realtime
└── Cron
```

### React + Vite

Клиентское приложение реализует интерфейс мессенджера, локальную работу с ключами и шифрование/расшифровку сообщений. Исходный код организуется по методологии FSD.

### Cloudflare Pages

Статический хостинг собранного React-приложения. Серверная бизнес-логика приложения на Cloudflare Pages не размещается.

### Supabase

- **Auth** — аутентификация и управление пользовательскими сессиями.
- **Postgres** — хранение пользователей, чатов, метаданных и зашифрованных сообщений.
- **Realtime** — доставка новых сообщений и обновлений клиентам в реальном времени.
- **Cron** — периодический запуск автоматической очистки сообщений старше 180 дней.

## Архитектурные ограничения

- Собственный backend и отдельный сервер приложений не используются.
- Supabase не должен получать открытый текст сообщений: шифрование и расшифровка выполняются на клиенте.
- В базе данных сохраняются только зашифрованное содержимое сообщения и необходимые технические метаданные.
- Приложение поддерживает только текстовые сообщения; вложения, изображения, аудио и видео не входят в текущий scope.
- Доступ к данным ограничивается политиками Supabase Row Level Security (RLS).
- Сообщения старше 180 дней автоматически и безвозвратно удаляются задачей по расписанию.
